title: 'MCP for agents'
description: 'Give your agent email as native tools over the Model Context Protocol.'
---

SentVia ships a **Model Context Protocol (MCP) server** so an AI agent can use email
as first-class tools — create inboxes, send and reply, search, manage webhooks —
without writing a single API call. The agent simply *calls a tool*.

## Why MCP

With the REST API your code orchestrates email. With MCP, the **agent itself**
decides when to check its inbox, draft a reply, or escalate — SentVia's operations
appear directly in the model's tool list.

## Connect

Point your MCP-capable client (Claude, or any MCP host) at the SentVia server and
authenticate with your API key:

```json
{
  "mcpServers": {
    "sentvia": {
      "command": "npx",
      "args": ["-y", "sentvia-mcp"],
      "env": { "SENTVIA_API_KEY": "sv_live_…" }
    }
  }
}
```

<Note>
  The exact server command/endpoint for your workspace is shown in the
  [dashboard](https://console.sentvia.ai). Authentication uses the same API keys as
  the REST API — see [Authentication](/authentication).
</Note>

## Available tools

The server exposes the core SentVia operations as tools, mirroring the API:

- **Inboxes** — create, list, delete
- **Messages** — send, reply, forward, get, search
- **Threads** — list, read a conversation
- **Webhooks & lists** — manage delivery endpoints and allow/block rules

Because the tools map onto the same endpoints documented in the
[API reference](/api-reference/introduction), everything you read here applies —
threading, idempotency, suppression, and attachments all behave identically.
